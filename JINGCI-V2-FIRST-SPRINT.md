# 镜词 V2 第一阶段任务清单

## Sprint 目标
完成镜词 V2 的产品骨架升级，让产品从 prompt 工作台进入导演执行包模式。

## P0 必做
### 1. 重新定义首页流程
- 输入创意
- 创意体检
- 选择重构方向
- 生成导演执行包

完成定义：有清晰页面结构和字段定义。

### 2. 定义创意体检输出结构
字段至少包括：
- 可拍性评分
- 风险点
- 改造建议
- 推荐方向

完成定义：前后端都知道该返回什么。

### 3. 定义类型重构输出结构
至少输出 3 个版本：
- 最稳出片版
- 最强风格版
- 最像短片版

完成定义：每个版本有解释，不只是换词。

### 4. 定义导演执行包结果结构
结果必须包含：
- 创意诊断
- 重构版本
- 分镜卡片
- 每镜生成策略
- 平台建议
- 后期建议
- 风险补救

完成定义：结果页结构可直接进入 UI 设计。

### 5. 镜头风险标签体系
至少先定义：
- 低风险
- 中风险
- 高风险

并标注触发原因：
- 多人同框
- 口型依赖
- 复杂人脸一致性
- 快速动作失真

完成定义：每镜头都能挂风险标签。

## P1 应做
### 6. 模板策略
先定义 3 套：
- 废土科幻
- 黑色幽默
- 赛博惊悚

### 7. 平台建议模块
按镜头或整体给出：
- 更适合 Seedance
- 更适合 Kling
- 更适合图生视频

### 8. 后期建议模块
给出：
- 剪辑节奏
- 音效建议
- 字幕/旁白建议

## P2 可后移
### 9. 用户反馈闭环
### 10. 知识自动注入
### 11. 分享和传播机制

## 依赖关系
1. 先有输出结构
2. 再有页面结构
3. 再进 UI 与实现
4. 模板策略和平台建议可并行补充

## 风险
- 输出结构过重，页面会乱
- 类型重构可能流于表面
- 平台建议若没有真实生成经验支撑，会显得虚

## 验收标准
- 用户一眼能看懂新的流程
- 输出不是 prompt 堆砌，而是执行方案
- 结果页能指导下一步生成和剪辑
- 至少有 3 套清晰可复用的类型模板
