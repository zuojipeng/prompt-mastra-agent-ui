# 镜词 · 3 分钟 Demo 讲解脚本

## Demo 目标

展示镜词如何把一句 AI 视频创意变成可执行导演包，并继续跟踪每个镜头的出片状态。

## 讲解结构

### 0:00 - 0:20 开场

口播：

```text
镜词不是普通 Prompt 生成器，而是 AI 短片导演执行包工作台。它解决的问题是：一个创意到底怎么更稳地做成片。
```

画面：
- 打开首页。
- 展示输入框和目标时长、类型选择。

### 0:20 - 0:50 输入创意

输入：

```text
废土小镇里，一个旧清洁机器人守护红裙人偶。
```

口播：

```text
我们不用一上来就复制 Prompt，而是先让系统判断这个创意是否适合 AI 视频生成。
```

操作：
- 选择 60s。
- 选择赛博朋克。
- 点击“先做创意体检”。

### 0:50 - 1:20 创意体检

口播：

```text
体检会告诉我们可拍性评分、关键风险和推荐改造方向。这里的重点是提前发现主体一致性、复杂动作这类常见翻车点。
```

操作：
- 展示创意体检报告。
- 点击“查看重构版本”。

### 1:20 - 1:50 选择重构版本

口播：

```text
镜词会给三个方向：稳妥版、风格版、电影版。实际创作建议先用稳妥版跑通，再追求风格和电影感。
```

操作：
- 展示三个版本。
- 选择电影版。
- 点击“用此版本生成执行包”。

### 1:50 - 2:30 导演执行包

口播：

```text
执行包不是一段 Prompt，而是一套短片生产清单：故事设定、分镜、每镜生成策略、主 Prompt、平台建议、后期建议和风险补救。
```

操作：
- 展示故事设定。
- 展示分镜卡片。
- 展示风险标签和稳定性检查。

### 2:30 - 2:50 逐镜头执行

口播：

```text
真正出片时，我们按镜头执行。每个镜头生成后，可以标记已生成、翻车或可用，进度会回到执行包顶部。
```

操作：
- 点击“翻车”。
- 再点击“可用”。
- 展示出片执行进度变化。

### 2:50 - 3:10 反馈洞察

口播：

```text
最后，用户反馈会进入反馈洞察，帮助我们知道哪些平台、镜头类型和失败原因最值得优化。
```

操作：
- 打开“反馈洞察”。
- 展示高频失败原因和最近差评样本。

### 3:10 - 3:20 收尾

口播：

```text
这就是镜词当前的产品形态：从创意判断，到导演执行包，再到逐镜头出片和反馈闭环。
```

## 画面检查清单

- 首页输入框清晰可见。
- 创意体检报告清晰可见。
- 三个重构版本清晰可见。
- 分镜卡片、风险标签、稳定性检查清晰可见。
- 出片执行进度可以变化。
- 反馈洞察可以打开。
