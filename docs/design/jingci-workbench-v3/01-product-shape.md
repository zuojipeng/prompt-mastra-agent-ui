# 01 · Product Shape

## Product Shape

Jingci is a professional AI short-film director workbench.

It is not:
- a generic prompt generator
- a landing page
- a model playground
- a template gallery
- a social content platform

## User Job

```text
When I have an AI video idea but do not know how to make it generate reliably,
I want to turn it into a shot-by-shot director execution package,
so I can produce usable clips with less platform trial-and-error.
```

## Primary User

Independent AI video creator using tools such as Seedance, Kling, Runway, Pika, Sora, or XiaoYunque.

The user is comfortable copying prompts into external platforms, but needs help with:
- feasibility judgment
- shot decomposition
- platform fit
- risk avoidance
- execution tracking
- feedback learning

## Experience Promise

Jingci should make the user feel:

```text
I am directing a short film with a production plan,
not guessing prompts in a blank box.
```

## Design Principles

1. **Decision first**
   Every region should help the user choose, execute, or recover.

2. **Workbench over feed**
   The interface should support repeated work, comparison, and status tracking.

3. **Risk is a first-class object**
   Risks should be visible near the shot or platform decision they affect.

4. **Execution evidence matters**
   Generated links, status, notes, and feedback belong close to the shot.

5. **Professional density**
   Use compact hierarchy, not oversized marketing composition.

6. **No hidden critical state**
   Current project, stage, progress, and next action must remain visible.

## North Star Behavior

User starts with one idea and completes at least one platform-ready shot execution loop:

```text
idea -> DirectorKit -> copy shot prompt -> generate externally -> record result -> submit feedback
```
