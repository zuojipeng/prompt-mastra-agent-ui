# 镜词 · 升级规则

## 目标
当 Hermes 或子 Agent 卡住、跑偏、需要确认时，不允许静默等待，必须立刻暴露问题。

## 升级类别

### 1. blocked_interactive_confirmation
触发条件：
- 命令要求输入 yes/no
- 出现登录、授权、菜单选择
- 出现覆盖确认、删除确认

处理动作：
- 立即停止继续等待
- 在状态文件中标记 `blocked_interactive_confirmation`
- 写入阻塞原因、所需人工动作、影响范围
- 触发即时汇报，不等 10:00 / 22:00

### 2. blocked_authentication
触发条件：
- Codex / OAuth / API key 缺失或失效
- 第三方登录被占用或过期

处理动作：
- 标记 `blocked_authentication`
- 说明受影响任务
- 给出替代路径或降级方案
- 无替代路径时立刻升级

### 3. blocked_human_decision
触发条件：
- 需求冲突
- 产品方向不清
- 是否上线、是否改系统配置等需要老板拍板

处理动作：
- 标记 `blocked_human_decision`
- 生成待决策事项
- 给出推荐方案 A/B

### 4. stalled
触发条件：
- 单步骤 10 分钟没有状态变化
- 命令 2 分钟没有新输出且疑似等待
- 任务超过预计时间 2 倍

处理动作：
- 标记 `stalled`
- 记录最后有效进展时间
- 触发即时汇报

### 5. quality_gate_failed
触发条件：
- 验收门未通过
- 文档缺失
- 交付不完整

处理动作：
- 直接打回上一步
- 记录失败原因
- 重新给出完成定义

## 强制规则
- 到点未汇报，视为执行失败
- 遇到交互确认，不准挂着等
- 需要人工确认时，不准伪装成“仍在执行”
- 同一阻塞连续出现 2 次，必须升级到老板可见层

## 升级输出模板
- 当前状态
- 阻塞类型
- 影响任务
- 需要人工做什么
- 推荐下一步
- 若不处理的风险
