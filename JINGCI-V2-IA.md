# 镜词 V2 信息架构稿

## 目标
把镜词从“生成 prompt 的页面”改成“引导用户完成一次 AI 短片创作决策”的页面。

---

## 一、首页信息架构

### 区块 1：顶部价值说明
目标：让用户一眼知道这不是普通 prompt 工具。

建议文案：
- 标题：把一句创意，变成更容易出片的 AI 短片执行包
- 副标题：先判断值不值得拍，再重构，再生成导演执行包

### 区块 2：创意输入区
字段：
- 创意输入框
- 目标时长（15s / 30s / 60s）
- 类型目标（废土 / 赛博 / 黑色幽默 / 惊悚 / 自定义）
- 平台偏好（Seedance / Kling / Runway / 通用）

操作：
- 按钮 1：先做创意体检
- 按钮 2：直接生成（高级用户入口）

### 区块 3：创意体检结果
展示内容：
- 可拍性评分
- 3 个主要风险点
- 3 个改造建议
- 推荐方向

展示形式：
- 分数卡
- 风险标签
- 建议卡片

### 区块 4：重构方向选择
给 3 个版本卡片：
- 最稳出片版
- 最强风格版
- 最像短片版

每张卡片显示：
- 一句话摘要
- 为什么这么改
- 适合什么场景
- 风险降低了什么
- 按钮：用这个版本生成执行包

### 区块 5：模板捷径
放 3 套高价值模板入口：
- 废土科幻
- 黑色幽默
- 赛博惊悚

目的是让用户快速进入高胜率模式。

---

## 二、结果页信息架构

### 结果页总原则
结果页不再围绕“prompt 文本”展开，而是围绕“执行决策”展开。

### 区块 1：创意诊断总览
显示：
- 原始创意
- 选择的重构方向
- 可拍性评分
- 总风险等级
- 推荐平台

### 区块 2：重构后故事版本
显示：
- 重构后的一句话故事
- 导演意图说明
- 主角设定
- 世界观设定
- 固定视觉符号

### 区块 3：分镜卡片区
每一镜一张卡，建议字段：
- 镜头编号
- 时长
- 镜头目的
- 景别
- 画面描述
- 动作描述
- 情绪/氛围
- 运镜建议
- generationMode（文生 / 图生 / 参考图）
- consistencyNeed（低 / 中 / 高）
- riskLevel（低 / 中 / 高）
- fixSuggestion（翻车时怎么补）

### 区块 4：主执行 prompt 区
显示：
- 整体版本主 prompt
- 负向词
- 平台通用注意事项

### 区块 5：平台适配区
每个平台一张卡：
- 平台名
- 推荐 prompt
- 使用说明
- 限制提醒
- 是否推荐本次使用

### 区块 6：后期建议区
显示：
- 剪辑节奏建议
- 音效建议
- 配乐建议
- 字幕/旁白建议

### 区块 7：风险补救区
显示：
- 本次最容易翻车的 3 个点
- 替代镜头建议
- 重新生成建议
- 哪些镜头建议直接换成背影/远景/剪影

---

## 四、用户流程
1. 输入创意
2. 看创意体检
3. 选一个重构方向
4. 查看导演执行包
5. 逐镜生成
6. 按后期建议完成成片

这个流程的重点不是“快”，而是“少走弯路”。

---

## 五、和当前版本的最大变化
1. 首页从输入即生成，改成输入 -> 体检 -> 重构 -> 执行包
2. 结果页从 prompt 为中心，改成导演执行决策为中心
3. 分镜不只是描述镜头，还提供生成方式和风险补救
4. 平台适配不只是导出版本，而是给出推荐与限制

---

## 六、第一版实现建议
先做最小版本：
- 创意体检
- 3 个重构方向卡片
- 导演执行包结果页
- 分镜卡字段扩展
- 后期建议区

先别急着做：
- 社区
- 云同步
- 复杂分享
- 自动学习闭环

## 七、一句话总结
镜词 V2 的页面，不该让用户感觉自己在“写 prompt”，而应该让用户感觉自己在“做一条更容易出片的短片决策”。
