# 镜词 V2 执行版 PRD

## 1. 产品目标
把镜词从“提示词工坊”升级为“AI 短片导演工作台”。

第一阶段目标不是做大而全，而是让用户拿到一份更适合出片的导演执行包，减少 AI 视频创作里的低质量试错。

## 2. 第一阶段核心问题
当前产品能生成 prompt，但还不能很好回答：
- 这个创意值不值得拍
- 它哪里容易翻车
- 应该往哪个类型方向重构
- 每一镜更适合怎么生成
- 最终怎么更像一条短片，而不是一堆 prompt

## 3. 第一阶段范围
### In Scope
1. 创意体检
2. 类型重构
3. 导演执行包结果页
4. 镜头生成策略
5. 三套高价值模板

### Out of Scope
1. 用户系统
2. 团队协作
3. 排行榜/画廊
4. 商业化付费
5. 云端同步
6. 自动学习闭环

## 4. 功能模块
### 4.1 创意体检
输入一句创意后，返回：
- 可拍性评分
- 风险点
- 不推荐点
- 推荐改造方向

#### 示例字段
- feasibilityScore
- keyRisks
- suggestedAdjustments
- recommendedDirection

### 4.2 类型重构
给用户 3 个重构版本：
- 最稳出片版
- 最强风格版
- 最像短片版

每个版本都要解释：
- 为什么这么改
- 更适合什么平台
- 风险降低了什么

### 4.3 导演执行包
结果页输出：
1. 创意诊断
2. 重构后故事版本
3. 世界观与主角设定
4. 分镜卡片
5. 每镜生成建议
6. 平台适配
7. 后期剪辑建议
8. 风险补救建议

### 4.4 镜头生成策略
每一镜至少带：
- shotPurpose
- cameraDistance
- motion
- generationMode
- consistencyNeed
- riskLevel
- fixSuggestion

### 4.5 模板库
第一阶段先做 3 个：
- 废土科幻
- 黑色幽默
- 赛博惊悚

## 5. 页面流程
### 首页
步骤：
1. 输入创意
2. 查看创意体检
3. 选择重构方向
4. 生成导演执行包

### 结果页
分区：
- 创意体检
- 重构版本
- 分镜与生成策略
- 平台适配
- 后期建议
- 风险补救

## 6. 成功标准
### 产品成功标准
1. 用户更快判断创意能不能拍
2. 用户更少生成明显高风险镜头
3. 用户拿到的是执行方案，不只是 prompt
4. 用户更接近做出成片，而不是只复制文本

### 体验成功标准
1. 首页流程比现在更清晰
2. 结果页信息层次不乱
3. 用户能看懂为什么这样重构
4. 用户知道下一步该怎么生成

## 7. 风险
1. 信息太多，结果页可能过重
2. 体检建议如果太抽象，用户不信
3. 三个重构版本如果差异不够大，会显得价值弱
4. 平台适配若没有真实经验支撑，容易空泛

## 8. 第一阶段交付物
1. 新首页信息架构
2. 创意体检数据结构
3. 类型重构数据结构
4. 导演执行包结果结构
5. 模板策略文档
6. UI 草图 / 页面区块设计

## 9. 第一阶段任务拆解
### P0
- 定义创意体检输出结构
- 定义类型重构输出结构
- 定义导演执行包输出结构
- 设计新首页流程
- 设计新结果页模块

### P1
- 设计 3 套模板
- 增加镜头风险标签
- 增加后期建议模块
- 增加平台推荐说明

### P2
- 做用户反馈收集准备
- 做结构化状态埋点准备

## 10. 最终判断
第一阶段不是做更多，而是做更准。
镜词要从“帮你写 prompt”，升级成“帮你更稳地把片子做出来”。
